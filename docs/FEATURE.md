## 1. 센터 관리
- ✅ 센터 목록/추가/이름변경/삭제/순서변경 (사이드바)
- ✅ 5개 센터: KB손보부천(`kbsonhae`), KB손보정비(`kbjeongbi`), LG전자AS(`lge`), LG전자성수기(`lge_seongsu`), 평택시청(`pyeongtaek`)
- ✅ (2026-07-12) 상단 고정 탭(대시보드/데이터입력/이슈및히스토리/TO및목표값설정) 제거 → 사이드바에서 현재 선택된 센터 바로 아래에 서브메뉴로 노출 (`MAIN_TABS`, `renderCenterSubmenu()`). 센터를 바꾸면 서브메뉴는 자동으로 해당 센터 것으로 교체됨.
- ✅ (2026-07-12) 센터 전환(사이드바 클릭 또는 비밀번호 인증 성공) 시 화면상태 초기화(`resetDashboardStateForNewCenter()`): 탭=대시보드, 기간=단월, 월=이번달로 리셋. 이전엔 직전 센터에서 보던 탭·기간이 그대로 유지되는 버그가 있었음.
- 검증: 사이드바에서 센터 클릭 → 비밀번호 프롬프트(비관리자) 또는 즉시 진입(관리자) 확인. 서브메뉴 탭 전환 시 다른 센터 서브메뉴가 사라지고 새 센터 서브메뉴만 보이는지 확인.

## 2. 데이터입력 — 실적
| 센터 | 방식 | 상태 |
|---|---|---|
| KB손보부천 | 엑셀 자동추출(고정 필드) | ✅ |
| KB손보정비 | RAW 엑셀 자동추출 | ✅ |
| 평택시청 | HWPX 일일보고 자동추출 | ✅ |
| LG전자AS | 엑셀 자동추출 (summary 시트, **상시** Total) | ✅ 실제 파일 검증 완료 |
| LG전자성수기 | 엑셀 자동추출 (summary 시트, **한시** Total) | ✅ AS와 동일 로직, targetGroup만 다름 |
| 공통 | 직접 붙여넣기(여러 날짜 한번에) | ✅ |

- 파일 추출 완료 시 "직접입력" 화면으로 자동 전환 + 미리보기 자동 표시 (`selectEntryMethod('paste')` + `parseMultiPaste()`)
- 저장은 `manual-entry-bulk` 액션으로 **한 번에 일괄 저장** (5건씩 나눠 보내던 방식 폐기)
- 검증: 실제 KPI 파일로 추출 → 미리보기 표 숫자가 원본과 일치하는지 확인

## 3. 데이터입력 — 근태(재직/투입현황)
- ✅ KB손보정비/LG전자AS/LG전자성수기: 근태파일 업로드 시 자동 추출 (성명/업무/입사일/퇴사일 헤더 + `YYMM` 월별 시트 인식)
- ✅ 파일 선택 즉시 자동 추출 (`onchange="extractAttFile()"`)
- ✅ 이미 지정된 기간(실적 추출에서 넘어온 기간)이 있으면 덮어쓰지 않고 유지
- ✅ (2026-07-12) KB손보정비/LG전자AS/LG전자성수기: 상담사 투입인원(수동입력값) 0명인 날짜는 공휴일 여부와 무관하게 저장에서 자동 제외 (`skipZeroManual`, `saveAttRangeRows`/`saveAttAndPerfCombined` 양쪽 적용). 이전엔 LG전자AS/성수기만 적용되고 KB손보정비는 빠져 있었음.
- ✅ 기본 TO: LG전자AS `19`(관리자4+상담사15), LG전자성수기 `35`(공식, 관리자0+상담사35) / 운영기준 재직인원 기본값 `41`(관리자6+상담사35)
- 검증: 실제 근태파일(`2026_LG전자_AS센터_근태현황.xlsx`)로 6월 21영업일 전량 매칭 확인됨(첨부 파일로 검증한 이력 있음 — `CHANGELOG.md` 참고). KB손보정비는 투입인원 0명 날짜가 포함된 실제 파일로 재검증 필요.

## 4. 워크스페이스 전체현황 (홈)
- ✅ 상단 카드: 전체 센터 수 / 전체 월평균 재직인원 / 주의 센터 수(이상징후 감지)
- ✅ 이상징후 판정 기준: "마지막 저장일"이 아니라 **연초~최신월 누적 일평균**(주말·공휴일 제외) 기준
- ✅ 핵심지표 표시: 이번달 값 + 누적값 함께 표기, 지표별 단위(%/건/duration) 정확히 반영
- ✅ 센터별 추이그래프: 회색계열 통일(단, LG전자는 센터별 대시보드에서만 LG 브랜드컬러 사용)
- ✅ 🚦 센터 실적 업데이트현황 카드 (신호등, 섹션 8 참고)
- 검증: 워크스페이스 진입 → 핵심지표 옆 "(누적 xx)" 괄호값이 실제 연초 데이터 반영하는지 확인

## 5. 센터별 대시보드
- ✅ 요약카드(전체 운영요약 등) — 상단 기간토글(단월/분기누적/반기누적/연초누적)과 연동. `renderSummaryCards()`/`renderKbjeongbiSummaryCards2()` 모두 토글로 필터링된 `targetRows`를 받아 계산하므로, **전 센터 공통으로 이미 토글에 맞춰 카드 값이 바뀐다** (2026-07-12 코드 재검증 완료 — 별도 수정 불필요, 다른 값으로 보인다면 캐시/재접속 문제일 가능성이 높음).
- ✅ 지표추이 미니차트 — LG전자: 생산성(IN) + 생산성(OUT, 파생값) 누적막대 + T-NPS 선(LG RED)
- ✅ (2026-07-12) KB손보정비 주요지표(우측 패널)에 "전체재직인원(TO대비)" 바로 아래 "상담사 투입현황"(`상담사_투입인원`, people 타입) 추가
- 검증: 기간토글 클릭 시 카드 숫자가 실제로 바뀌는지 확인 (과거 버그였던 지점)

## 6. TO 및 목표값설정
- ✅ 연도별 1~12월 가로 표, 체크박스 일괄반영
- ✅ 핵심지표 월별 목표치 오버라이드 (`center_kpi_monthly_targets`)
- ✅ (2026-07-12) LG전자AS/성수기의 통화시간(`통화시간_INOUT_초`) 등 duration 지표는 목표값을 `4:00:00` 형식(시:분:초)으로 입력 가능 (`DURATION_METRIC_KEYS`, `parseKpiTargetInput()`/`formatKpiTargetDisplay()`). 저장은 항상 초 단위 숫자, 화면 표시만 H:MM:SS로 변환. 기본목표값/월별그리드/일괄반영 입력창 모두 적용.

- ✅ LG전자AS/성수기: 실적+근태 완전자동 (서버측 파싱 로직 포팅 완료)
