- 저장은 `manual-entry-bulk` 액션으로 **한 번에 일괄 저장** (5건씩 나눠 보내던 방식 폐기)
- ✅ (2026-07-13) KB손보부천/KB손보정비: 엑셀 자동추출에 추출 시작일/종료일 지정 가능 (비워두면 기존처럼 파일 내 전체 기간).
- ✅ (2026-07-13) KB손보부천: 근태(총원~제외보유계약)·실적(인입호~) 컬럼이 한 표에 같이 있어, 미리보기 표에서 "근태만 저장"/"실적만 저장"을 선택해 한쪽만 반영 가능 (`saveAllRows(groupFilter)`) — 기존에 잘 들어간 근태는 그대로 두고 실적만 다시 반영하는 식의 부분 수정용.
- ✅ (2026-07-17 4차) **KB손보정비: 실적/근태 완전 분리**. 이전엔 엑셀 자동추출 후 `promptGoToAttendance()`가 자동으로 "재직 및 투입현황" 탭으로 이동시키고 그 화면에 "실적+근태 일괄 반영" 버튼까지 노출돼 있어 두 데이터를 한 번에 묶어 저장하기 쉬운 구조였음. 이제 KB손보정비는 엑셀 자동추출/붙여넣기가 끝나도 자동 이동하지 않고 "일자별 실적 직접입력" 화면에 머무르며, 사용자가 그 화면의 "전체 저장"으로 **실적만** 먼저 저장한다. "재직 및 투입현황" 화면은 KB손보정비에 한해 "근태만 저장" 버튼만 노출되고(`실적+근태 일괄 반영`/`실적만 저장` 버튼은 숨김), 그 화면에서는 **근태만** 저장된다. 두 저장 모두 자기 그룹(`group: 'attendance'` 또는 `'performance'`) 필드만 전송하므로, 실적을 먼저 올리고 근태를 나중에 올리거나 그 반대여도 서로의 값을 지우지 않는다(기존에 검증된 `manual-entry-bulk`의 부분 필드 병합 동작에 근거). LG전자AS/성수기는 기존 방식(일괄 반영 버튼 포함) 그대로 유지.
- ✅ (2026-07-17 4차) **KB손보정비: "목적물소멸" 건수 판정 방식 수정**(`lcClassify()`). 기존엔 의무위반사항이 정확히 "단순변경"이고 처리구분에 "목적물소멸"이 포함된 경우만 인정했음. 이제는 의무위반사항/처리구분 **둘 중 어느 값에든 "목적물소멸"이라는 단어가 포함**돼 있으면 무조건 목적물소멸 1건으로 집계한다 — "목적물소멸,담보소멸", "담보납입면제,목적물소멸,담보무효취소"처럼 다른 사유와 콤마로 같이 적힌 값도 전부 포함됨. ⚠️ 실제 엑셀 파일로 검증하지 못했음(AGENTS.md "추측 금지" 원칙 예외 — 파일이 없어 두 컬럼 모두에 안전하게 적용하는 방식으로 구현) — 다음 실제 파일 업로드 시 집계 건수가 늘어난 게 맞는지 확인 필요.
- ✅ (2026-07-17 4차) **평택시청: "일자별 실적 직접입력"/"업무유형별 현황" 모두에 항목별 직접입력 표 추가**(LG전자통합과 동일한 형태: 날짜범위 지정 → 일괄입력 행 + 항목별/전체 "전체반영" + 날짜별 "제외" 체크박스). 기존 텍스트 붙여넣기 UI는 그대로 유지되고, 이 표는 그 아래에 별도 패널로 추가된 완전히 독립적인 입력 경로(저장 버튼도 따로)다. 컬럼은 하드코딩하지 않고 서버가 내려주는 `rowSchema`/`categorySchema`를 그대로 사용(`renderPtDailyItemShell`/`renderPtCategoryItemShell`). 일자별 실적 쪽은 "투입인원"(근태)만 직전 저장일 값으로 이월되고 나머지(요청호/응답호/응대율/포기호/CPD)는 매일 새로 입력. 업무유형별 현황 쪽은 이월 없이 매일 새로 입력하며, 저장 시 "합계" 대비 비중(%)을 자동 계산해 `{key}_비중` 키로 함께 저장(`savePtCategoryItemRows`, 기존 `saveCategoryRows`와 동일한 계산식).
- 검증: 실제 KPI 파일로 추출 → 미리보기 표 숫자가 원본과 일치하는지 확인. 기간 설정 필터링, KB손보정비 실적/근태 분리 저장(순서 바꿔가며), 목적물소멸 집계, 평택시청 항목별 직접입력 두 표 모두 실제 파일·저장 데이터로 재검증 필요.

- ✅ 이미 지정된 기간(실적 추출에서 넘어온 기간)이 있으면 덮어쓰지 않고 유지

- ✅ (2026-07-13) LG전자AS/LG전자성수기: "근태만 저장"/"실적+근태 일괄 반영"에 이어 "실적만 저장" 버튼 신규 추가(`saveAttPerfOnly`). 근태 기간과 겹치는 날짜의 실적만 반영하고 근태(TO/재직인원/투입인원)는 건드리지 않음.
- ✅ (2026-07-17 4차, 위 섹션2 "실적/근태 완전 분리" 참고) KB손보정비는 이 화면에서 "근태만 저장" 버튼만 노출됨 — "실적+근태 일괄 반영"/"실적만 저장"은 숨김 처리(`generateAttRangeRows()`의 `showCombinedAndPerfOnly` 분기). 자동추출 후에도 이 화면으로 자동 이동하지 않음(`promptGoToAttendance`가 kbjeongbi는 조기 반환).
- ✅ 기본 TO: LG전자AS `19`(관리자4+상담사15), LG전자성수기 `35`(공식, 관리자0+상담사35) / 운영기준 재직인원 기본값 `41`(관리자6+상담사35)
- 검증: 실제 근태파일(`2026_LG전자_AS센터_근태현황.xlsx`)로 6월 21영업일 전량 매칭 확인됨(첨부 파일로 검증한 이력 있음 — `CHANGELOG.md` 참고). KB손보정비는 투입인원 0명 날짜가 포함된 실제 파일로 재검증 필요. "실적만 저장"도 실제 저장 데이터로 근태 미변경 여부 재검증 필요.

## 7. 업로드 자료함
- ✅ 데이터입력 시 첨부한 원본 파일 자동 보관 (Storage 버킷 `uploaded-files`)
- ✅ 센터별 필터/파일명 검색/다운로드(서명URL)/개별삭제/일괄삭제
- ✅ (2026-07-13 버그수정) 파일을 선택/추출만 해도 즉시 자료함에 올라가던 문제 수정. 이제 **실제 저장이 성공한 시점에만** 업로드된다(`pendingPerfArchiveFile`/`pendingAttArchiveFile`). 추출만 하고 저장하지 않으면 자료함에 남지 않음.
- ✅ (2026-07-13 4차) 평택시청 일일업무보고(HWPX)는 원래 자료함 업로드 기능 자체가 없었는데, 다른 센터와 동일하게 "전체저장"(`saveEverything`) 성공 시에만 반영되도록 신규 추가.
- 검증: 파일을 추출만 하고 저장하지 않았을 때 자료함에 안 올라가는지, 저장 성공 후에는 올라가는지, 다운로드가 원본과 동일한지 확인. 평택은 일자별실적/업무유형별 인입호 중 한쪽만 저장됐을 때도 자료함에 반영되는지 별도 확인 필요.
